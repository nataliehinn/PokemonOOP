# PokemonOOP
AP Computer Science A Object-Oriented Programming final project resembling a single-player, text-based, Pokemon game where the main objective is to fight and catch pokemon. The user will be able to begin the game by choosing a starter Pokemon and then be prompted to choose coordinates on a map. The spot they land on may just be grass or could be a battle encounter. Before the battle, they will be able to choose which Pokemon they will be using to fight while in battle, the user will be able to either counter, attack, or defend. After they fight, their Pokemon will automatically heal allowing them to continue being prompted with other battles. The game continues until the user wants to quit.
